Case Study: Simulating Datacenter Temperature Distribution with OpenFOAM

OpenFOAM Journal Vol. 3 (2021)

Paper: Barestrand et al., "Datacenter CFD with OpenFOAM," OpenFOAM Journal v3 (2021), doi:10.51560/ofj.v3.59.

The Challenge

Reproduce Fig 3 of the paper, the room-temperature distribution in a datacenter, starting from the manuscript PDF and the author's OpenFOAM case bundle. Run the heavy CFD on a cloud cluster (not the laptop), then analyze the saved fields locally. Cap the SIMPLE solver at 500 iterations so cloud wall-clock stays under ~10 minutes; the KDE shape is the target, not residual convergence.

Prompt Given to SciAgent

Quick reproduction of Fig 3 (typical Boussinesq, 62K grid). Cap the SIMPLE solver at 500 iterations so wallclock stays under ~10 min on a small instance. The KDE shape is what matters, not residual convergence. Identify the OpenFOAM environment and boundary conditions from Manuscript.pdf and the case files in the project folder. When the solver finishes, pull the final time-step T field, the solver log, and the KDE plot back into the project folder. Then sky down the cluster, do this even on error.

What SciAgent Did Autonomously

  • Phase 1: Paper Analysis - Read manuscript and case bundle, extracted solver (buoyantBoussinesqSimpleFoam), turbulence model (k-epsilon), rack boundary conditions, supply temperature (291.45 K), and the volume-weighted KDE bandwidth from the author's plotting script. No human walkthrough of the physics.
  • Phase 2: Stage Modified Case - Wrote a coarse blockMeshDict (47x24x50, ~62k cells after snappy) and a 500-iteration controlDict; everything else preserved verbatim from the paper's steady_incompressible case.
  • Phase 3: Cloud Compute - Stood up a fresh compute cluster on demand, ran the meshing and solver pipeline on rented CPUs, materialized the result fields back to the project folder, and tore the cluster down afterwards. ~2.5 min wall-clock for 500 SIMPLE iterations.
  • Phase 4: Local KDE Analysis - Parsed the saved temperature and cell-volume fields (61,808 cells), computed the volume-weighted Gaussian KDE with the paper's bandwidth, and rendered the plot. Same fields can be re-analyzed without re-running the solver.
Paper + cases -> Stage local -> Cloud cluster | meshing -> solver (500 iter) -> T, V, logs | materialize -> local KDE -> Fig 3
