Case Study: Simulating Datacenter Temperature Distribution with OpenFOAM

OpenFOAM Journal Vol. 3 (2021)

Paper: Barestrand et al., "Modelling Convective Heat Transfer of Air in a Data Center using OpenFOAM — Evaluation of the Boussinesq Buoyancy Approximation," OpenFOAM Journal v3 (2021), doi:10.51560/ofj.v3.59.

The Challenge

Reproduce Fig 3 of the paper, the volume-weighted KDE of room temperature in a datacenter, starting from the manuscript PDF and the author's OpenFOAM case bundle. Run the heavy CFD on a cloud cluster (not the laptop), then analyze the saved fields locally. Cap the SIMPLE solver at 1000 iterations so cloud wall-clock stays under ~60 minutes on a small instance; the KDE shape is the target, not residual convergence.

Prompt Given to SciAgent

Reproduce Fig 3 (typical Boussinesq, 62K grid) from the manuscript on datacenter temperature distribution. Cap the solver at 1000 iterations so wall clock stays under ~60 min on a small instance. The KDE shape is what matters, not residual convergence. Identify the OpenFOAM environment and boundary conditions from Manuscript.pdf and the case files in the project folder. Pull all intermediate and final results back into the project folder.

What SciAgent Did Autonomously

  • Phase 1: Paper Analysis - Read manuscript and case bundle, extracted solver (buoyantBoussinesqSimpleFoam), turbulence model (k-epsilon), rack boundary conditions, supply temperature (291.45 K / 18.3 C), the "c" coarse 140 mm grid (47x24x50), and the volume-weighted KDE bandwidth from the author's plotting script. No human walkthrough of the physics.
  • Phase 2: Stage Modified Case - Wrote a coarse blockMeshDict (~56k background cells, ~62k after snappy) and a 1000-iteration controlDict; everything else preserved verbatim from the paper's steady_incompressible case.
  • Phase 3: Cloud Compute - Stood up a fresh AWS m6i.2xlarge (8 vCPU, 32 GB) cluster on demand, ran the meshing and 8-rank MPI solver pipeline, materialized the result fields and logs back via S3-backed workspace, and tore the cluster down afterwards. 258 s wall-clock for 1000 SIMPLE iterations.
  • Phase 4: Local KDE Analysis - Parsed the saved temperature and cell-volume fields (61,927 cells), computed the volume-weighted Gaussian KDE with the paper's bandwidth (covariance_factor = 0.1), and rendered the plot. Same fields can be re-analyzed without re-running the solver.
